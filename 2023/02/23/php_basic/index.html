如果循环结束后还未找到目标值，则返回 false。

示例

在 PHP 中，类的析构方法是一个特殊的方法，用于在对象被销毁之前进行一些清理工作。析构方法的名称必须是 __destruct()，它不接受任何参数，也不需要返回任何值。当对象被销毁时，PHP 自动调用该对象的析构方法。

析构方法通常用于释放在类的生命周期中分配的资源，例如打开的文件、数据库连接或分配的内存等。当 PHP 的垃圾回收机制检测到没有任何引用指向对象时，就会自动调用析构方法。

在 PHP 中，类的属性和方法可以使用访问控制符来限制其访问权限，这些访问控制符包括以下三种：

  1. public（公有）：类的外部和内部均可访问。
  2. protected（受保护的）：类的内部和子类中可访问，类的外部不可访问。
  3. private（私有的）：仅类的内部可访问，类的外部和子类均不可访问。

要定义一个抽象类，在类名前使用 abstract 关键字即可。在抽象类中定义的抽象方法也需要使用 abstract 关键字声明，但不需要包含方法体。子类必须实现抽象类中定义的所有抽象方法，否则该子类也必须被声明为抽象类。

抽象类的方法

抽象类可以包含抽象方法和非抽象方法。

抽象方法是没有实现的方法，其定义只包括方法名、参数和返回类型声明。抽象方法必须在抽象类中声明。任何继承抽象类的子类都必须实现其抽象方法。

非抽象方法是有实现的方法，可以在抽象类中直接实现。子类可以继承并重写非抽象方法。

接口是一种抽象类型，可以在接口中定义一组抽象方法，而不需要实现它们。接口的主要目的是定义规范，以便不同的类可以实现相同的行为。

在 PHP 中，使用 interface 关键字定义接口。接口中的方法默认都是抽象的，不需要使用 abstract 关键字。

注意：

以下是 PHP 中接口的一些注意事项：

  1. 接口中定义的所有方法都必须是公共方法（public），不能定义为私有方法（private）或受保护方法（protected）。
  2. 接口不能包含属性，只能包含方法的声明。
  3. 实现接口的类必须实现接口中所有的方法，否则会报错。
  4. 一个类可以同时实现多个接口，多个接口之间使用逗号（,）分隔。
  5. 接口也可以继承其他接口，通过使用 extends 关键字实现，可以继承多个接口。
  6. 如果一个类同时实现了多个接口，且这些接口中有同名的方法，那么这个类只需要实现一次该方法即可。
  7. 接口可以被类或抽象类实现，实现接口的类必须实现接口中所有的方法，而抽象类可以只实现部分方法，剩下的方法留给其子类去实现。
  8. 接口中的方法默认都是抽象的，不需要使用 abstract 关键字。
