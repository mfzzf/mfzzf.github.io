5. 其它场景

  • 也可用 Docker 的export/import。不过这样会丢失 Volume 挂载的数据和部分元信息，不推荐用于迁移重要服务。
  • 生产环境建议所有重要数据都用 Volume 持久化，避免直接将数据写入容器本地。

小结（简要步骤）

  1. 导出镜像：docker save
  2. 备份 volume：用 tar 归档
  3. 记录/导出容器配置
  4. 文件传输到目标主机
  5. 恢复镜像：docker load
  6. 恢复 volume：tar 解压
  7. 按配置重建容器

这样即可在新主机完整还原运行环境和数据。
